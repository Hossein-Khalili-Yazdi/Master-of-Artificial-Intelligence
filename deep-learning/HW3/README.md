# 🧠 2 Perceptrons for 4-Class Classification

A lightweight Python implementation showing how **two Single-Layer Perceptrons** can be combined from scratch to solve a **4-class classification** problem.
This project demonstrates how two simple artificial neurons, each learning one linear decision boundary, can be combined to carve the 2D plane into four separate regions.

---

## 📌 Features

* ⚙️ **From-Scratch Implementation:** Perceptron learning rule built without high-level deep learning frameworks.
* 🚪 **4-Class Coverage:** Two perceptrons combine their binary outputs (right/left, top/bottom) to correctly identify 4 classes — **C1, C2, C3, C4**.
* 📈 **Decision Boundary Visualization:** Includes automated Matplotlib plots showing both hyperplanes that partition the plane into four quadrant-like regions.
* 🎯 **Accuracy Verification:** Console-style verification confirming correct classification ($100\%$ accuracy) across all sample points.

## 🛠️ How it Works

Each perceptron computes the weighted sum of inputs plus a bias term, then applies a step activation function:

$$f(x) = \begin{cases} 1 & \text{if } w_1 x_1 + w_2 x_2 + b \ge 0 \\ 0 & \text{otherwise} \end{cases}$$

During training, weights and bias are updated according to the error rule:

$$w_i \leftarrow w_i + \eta (y - \hat{y}) x_i$$

$$b \leftarrow b + \eta (y - \hat{y})$$

Where:
* $\eta$ is the learning rate.
* $y$ is the target output.
* $\hat{y}$ is the predicted output.

The trick behind 4-class separation: two perceptrons each solve a simple binary sub-problem (right/left and top/bottom). Combining their two binary outputs — like 2 bits — uniquely identifies each of the 4 classes:

| Perceptron 1 (Right/Left) | Perceptron 2 (Top/Bottom) | Class |
|:---:|:---:|:---:|
| 1 | 1 | C1 |
| 1 | 0 | C2 |
| 0 | 1 | C3 |
| 0 | 0 | C4 |
